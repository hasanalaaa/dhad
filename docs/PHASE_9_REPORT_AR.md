# تقرير Phase 9 — API وSDK وLSP

## الإصدار

`v0.11.0`

## النطاق المنفذ

اكتمل النطاق المطلوب لهذه المرحلة: عقد REST إصداري، وSDK بايثون موحد للتشغيل المحلي والبعيد، وخادم LSP 3.17 للتكامل مع المحررات. لم يُنسخ أي منطق لغوي إلى هذه الطبقات؛ جميعها تستدعي نواة `Dhad` نفسها وتعيد نماذج موحدة قابلة للتحقق.

## REST API

- خادم FastAPI غير متزامن، مع نقل الأعمال اللغوية الثقيلة إلى thread pool حتى لا تُحجب حلقة الأحداث.
- نماذج Pydantic v2 صارمة (`strict=True`, `extra=forbid`) للطلبات والاستجابات.
- حد أقصى معلن للنص قدره مليون محرف لكل طلب JSON.
- المسارات المختصرة والإصداريّة:
  - `POST /check` و`POST /api/v1/check`
  - `POST /parse` و`POST /api/v1/parse`
  - `POST /diacritize` و`POST /api/v1/diacritize`
  - `POST /style` و`POST /api/v1/style`
  - `POST /dialect` و`POST /api/v1/dialect`
- توثيق آلي في `/docs` و`/redoc`، ومخطط OpenAPI في `/openapi.json`.
- بقي `POST /v2/check` و`GET /v2/languages` متوافقين مع العقد السابق لـLanguageTool.
- تدعم دالة `create_app()` حقن محرك مستقل للاختبارات أو الاستضافة متعددة الإعدادات.

## Python SDK

أضيف `DhadClient` في `dhad/client.py`:

- الوضع المحلي الافتراضي يستدعي النواة داخل العملية.
- الوضع البعيد يستخدم REST عبر مكتبة Python القياسية، من دون إضافة عميل HTTP ثقيل إلى التثبيت الأساسي.
- يعيد الوضعان نماذج Pydantic نفسها، فلا تتغير عقود المستهلك عند الانتقال من local إلى remote.
- واجهات متزامنة وغير متزامنة: `check/acheck`, `parse/aparse`, `diacritize/adiacritize`, `style/astyle`, `dialect/adialect`.
- أخطاء النقل أو JSON غير الصحيح تُغلّف في `DhadClientError` برسالة قابلة للتشخيص.

## LSP 3.17

أضيفت حزمة `dhad/lsp/` مع خادم stdio مستقل:

- `textDocument/didOpen`, `didChange`, `didSave`, `didClose`.
- مزامنة كاملة أو متزايدة وإعادة نشر `textDocument/publishDiagnostics` بعد كل تغيير.
- `textDocument/hover` يعرض اللمّة والجذر والوزن ونوع الكلمة وثقة الصرف والإعراب المرشح.
- `textDocument/codeAction` يعرض Quick Fix فقط عندما تكون نتيجة النواة `autofix=True` ولها بديل فعلي.
- لا تُعرض تحويلات اللهجات أو الأسلوب أو الاقتراحات العصبية كإصلاحات آمنة.
- تحويل دقيق بين offsets في Python ومواضع UTF-16 المطلوبة من LSP، بما في ذلك المحارف خارج BMP.
- تنفيذ JSON-RPC 2.0 بإطارات `Content-Length` وحد أقصى 64 MiB للرسالة.
- التشغيل عبر `dhad lsp` أو `dhad-lsp`.

- عدد اختبارات المشروع: **1,623 اختبارًا ناجحًا**.
- اختبارات Phase 9 الجديدة: **20** اختبارًا لعقود API وSDK وLSP.

- Ruff وفحص التنسيق: نظيفان.

- تم اختبار handshake فعلي لخادم LSP عبر subprocess وإطارات JSON-RPC.

## حدود الادعاء

- لم يُضف هذا الإصدار SDK TypeScript.
- لا توجد بعد واجهة REST لجلسات المستندات أو إلغاء الطلبات طويلة التنفيذ.
- مخرجات CLI بصيغتي SARIF وJSONL ليست ضمن هذا الإصدار.
- اختبار الحمل المتزامن على عتاد مرجعي منشور جزء من مرحلة الإنتاج اللاحقة.
