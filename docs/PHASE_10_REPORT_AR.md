# تقرير Phase 10 — الويب، الإضافة، سطح المكتب، والهاتف

## الإصدار

`v0.12.0`

## النطاق المنجز

أُنجزت طبقات العرض المطلوبة فوق عقود Phase 9 من دون نسخ أي منطق لغوي:

1. **Web/PWA**
   - محرر عربي RTL متجاوب للتدقيق والأسلوب واللهجات والتشكيل.
   - ملفات HTML/CSS/JS مستقلة قابلة للصيانة، مع معاينة ملوّنة وتطبيق اقتراحات صريح.
   - manifest قابل للتثبيت وأيقونات 192/512.
   - Service Worker يخزن ملفات shell فقط، ويتجاهل عمدًا جميع API requests/responses الحاملة للنص.

2. **تكامل FastAPI والأمان**
   - ملفات static اختيارية عبر `create_app(serve_web=True|False)` و`dhad serve --no-web`.
   - CORS افتراضي محصور في loopback، مع origins خاصة بإضافات Chrome/Firefox ورفض `*`.
   - CSP و`X-Frame-Options: DENY` وPermissions Policy و`nosniff`.
   - استمرار REST/OpenAPI/LanguageTool دون تغيير.

3. **إضافة Chrome/Edge — Manifest V3**
   - Service Worker موحد لكل الشبكة والإعدادات والمهلات.
   - أذونات مضيف دائمة محلية فقط، والخادم البعيد يحتاج موافقة `optional_host_permissions`.
   - Content Script يراقب `textarea` و`contenteditable`، ويرسم overlay بخطوط متموجة من دون تغيير النص.
   - بطاقة مراجعة تدعم الاستبدال مع تمييز `autofix` الآمن عن اقتراحات الأسلوب واللهجة.
   - Popup لإيقاف الإضافة وتغيير الخادم والمهلة وفئات العرض واختبار الاتصال.

4. **سطح المكتب والهاتف**
   - `dhad-desktop` و`dhad desktop` يديران دورة حياة Uvicorn على loopback فقط.
   - pywebview اختياري؛ fallback إلى Chrome/Edge/Chromium app mode ثم المتصفح الافتراضي.
   - PWA هي أساس الهاتف الحالي. لا تُدّعى لوحة مفاتيح أصلية أو حزمة متجر هاتف.

## التحقق

- 17 اختبارًا جديدًا لواجهات Phase 10.
- 1,642 اختبارًا ناجحًا عبر مجموعات المشروع.
- Node `--check` لكل JavaScript في الويب والإضافة.
- تشغيل فعلي لمشغّل سطح المكتب والتحقق من `/api/health` والصفحة الرئيسية.
- بناء واختبار Wheel/Source Distribution في بيئة مستقلة.
- Benchmark test الرسمي بقي: Precision 0.8602، Recall 0.9539، F0.5 0.8775، وFP/1000 = 6.3511.

## حدود الادعاء

- لم تُنشر الإضافة في متجر Chrome/Edge بعد.
- لم تُنفذ حزمة Firefox/Safari أو لوحة مفاتيح أصلية.
- لم تُنفذ اختبارات E2E مرئية على مصفوفة متصفحات؛ سياسة Chromium الإدارية في بيئة البناء منعت التنقل الشبكي، لذلك لم يُسجل ادعاء اختبار بصري.
- لا يوجد محرر مشاريع متعدد الملفات أو مزامنة حسابات في v0.12.0.
