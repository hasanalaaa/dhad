# تقرير Phase 11 — الإنتاج والأمان والخصوصية

## الإصدار

`v0.13.0`

## ما نُفّذ

### 1. نشر إنتاجي قابل لإعادة البناء

- `Dockerfile` متعدد المراحل: بناء Wheel واعتماداته في مرحلة منفصلة، ثم تثبيتها في صورة تشغيل صغيرة من دون أدوات البناء.
- تشغيل الحاوية بمستخدم غير جذري ثابت UID/GID `10001`.
- `HEALTHCHECK` فعلي على `/api/health`.
- `docker-compose.yml` يربط افتراضيًا على loopback، ويستخدم filesystem للقراءة فقط، و`tmpfs`، و`no-new-privileges`، وإسقاط جميع Linux capabilities.
- `gunicorn_conf.py` مع Uvicorn workers، preload للنواة، تدوير العمال، graceful timeout، وسجل وصول لا يحتوي query strings أو headers أو bodies.

### 2. طبقة الأمان HTTP

أضيف `dhad/security.py` وفيه:

- `SecuritySettings` متحقق منها وقابلة للضبط من متغيرات البيئة.
- Token Bucket متزامن آمن للطلبات، مع تنظيف دوري للحالات الخاملة ورؤوس `X-RateLimit-*` و`Retry-After`.
- مصادقة اختيارية بمفاتيح API عبر `X-API-Key` أو `Authorization: Bearer`، مع مقارنة hashes باستخدام `hmac.compare_digest`.
- حد افتراضي للنص: **50,000 حرف**.
- حد افتراضي لجسم الطلب: **262,144 بايت**، ويُطبق أثناء streaming حتى عند غياب `Content-Length`.
- ردود validation منقحة لا تعيد قيمة الإدخال الخاصة إلى العميل.
- `Cache-Control: no-store` لمسارات التحليل، إضافة إلى CSP وCOOP وCORP وHSTS عند HTTPS ورؤوس الحماية السابقة.

### 3. الخصوصية وقناع PII

أضيف `dhad/privacy.py` وفيه:

- كشف محافظ لعناوين البريد الإلكتروني وأرقام الهاتف والروابط.
- استبدال كل span بمحرف Private-Use فريد مكرر بالطول نفسه؛ لذلك تبقى كل offsets مطابقة للنص الأصلي.
- تشغيل الصرف والنحو والتدقيق العصبي والأسلوب واللهجات والدلالة والتشكيل على النسخة المقنعة فقط.
- استعادة PII حرفيًا في النتائج النصية حتى عندما يضيف التشكيل حركات قبل الموضع المحمي.
- تصفية أي diagnostic يتداخل مع PII.
- تطبيق الحماية داخل `Dhad` نفسه، لذلك تشمل REST وSDK وCLI وLSP، لا الخادم فقط.
- سياسة zero-text logging: لا يوجد logging لأجسام الطلبات، مع `PrivacyLogFilter` دفاعي لتنقيح PII من السجلات العرضية.

## التحقق

- **1,659 اختبارًا** في المجموعة النهائية، منها 17 اختبارًا جديدًا للأمان والخصوصية والنشر.
- Ruff وformat و`compileall` نظيفة.
- Benchmark الكامل مطابق لـv0.12.0:
  - Precision: `0.8602`
  - Recall: `0.9539`
  - F0.5: `0.8775`
  - FP/1000: `6.3511`

## حدود الادعاء

- Token Bucket ذاكرته محلية لكل process؛ النشر متعدد العقد يحتاج Redis أو مخزنًا موزعًا.
- مفاتيح API ثابتة من البيئة وليست نظام multi-tenant أو OAuth/JWT أو إدارة أسرار كاملة.
- القناع regex عالي الدقة للأنماط المطلوبة، لكنه ليس DLP شاملًا ولا يكتشف الأسماء والعناوين البريدية والهوية الوطنية.
- لم يُنفذ load test معلن، أو pentest مستقل، أو SBOM وتوقيع supply-chain في هذه النسخة.
- Queue منفصلة للمهام العصبية وSLO/P95/P99 وbackup/restore تبقى ضمن النطاق الموسع قبل تشغيل خدمة عامة واسعة.
