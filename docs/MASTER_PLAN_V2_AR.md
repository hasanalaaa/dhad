**المبدأ الحاكم:** كل مرحلة تحافظ على العقيدة المؤسِّسة — حتمي أولًا، خصوصية أولًا، لا إصلاح تلقائي غير آمن، كل شيء قابل للتفسير — وتُسلَّم بكود إنتاجي واختبارات خضراء، لا نماذج أولية.

## V2-م1 — بصمة الأسلوب والصوت الشخصي (تُنفَّذ الآن)

الفجوة: المحرك يعامل كل الكتّاب كاتبًا واحدًا. الحل: `dhad.stylometry` — استخلاص حتمي قابل للتفسير لبصمة أسلوب الكاتب من نصوصه (طول الجمل وتوزيعها، الثراء المعجمي، عادات الترقيم، أدوات الربط المفضلة، كثافة التشكيل، الميول الإملائية)، ملف صوت (VoiceProfile) قابل للحفظ JSON بنسخة مخططة، ومقارنة انحراف أي نص جديد عن البصمة مع **شرح رقمي لكل بُعد**.

قيود صلبة: التعلم يمر عبر قناع PII قبل أي استخلاص؛ البصمة أرقام مجمّعة لا نصوص خام (لا يمكن إعادة بناء النص من الملف)؛ الشخصنة تعيد ترتيب ملاحظات *الأسلوب* وتشرحها ولا تكتم خطأً إملائيًا/نحويًا أبدًا. هذه حجر أساس «استنساخ الصوت» لاحقًا: الكتابة بصوت المستخدم تتطلب أولًا قياس صوته.

**التسليم:** `src/dhad/stylometry.py` + ربط آمن في `Dhad` (`learn_voice` / `voice_report`) + اختبارات شاملة، دون أي تغيير في سلوك `check()` الحالي.

## V2-م2 — النواة التزايدية (إصلاح عنق الزجاجة قبل أي نقل لغة)

- استبدال مرشّح التداخل التربيعي بـ sweep-line على offsets: O(M log M).
- تجزئة المستند إلى جمل ذات هوية مستقرة (hash محتوى + سياق) مع مخبأ نتائج لكل جملة؛ ضغطة المفتاح تعيد تحليل الجملة المتغيرة وجاراتها فقط.
- مسار `apply_single_match` رخيص للمحررات، وworker pool منفصل عن طبقة HTTP مع `preload_app` وCOW.
- هدف مقيس: p50 < 15ms لضغطة مفتاح في مستند 10k كلمة، دون أي تغيير في نتائج الفحص الكامل (اختبارات تكافؤ بت-ببت مع المسار الحالي).

## V2-م3 — النواة المحمولة وWASM (الحافة والمتصفح)

- استخراج «النواة الحتمية الدنيا» (tokenizer، normalize، محرك القواعد، الفحوص المدمجة، Safe-Autofix) إلى **مواصفة قابلة للنقل** مع مجموعة اختبارات ذهبية مشتركة (JSONL: نص → ملاحظات) تكون عقد التوافق بين التطبيقات.
- تنفيذ Rust لهذه النواة (`dhad-core-rs`) يُترجم إلى WASM (< 2MB مضغوطًا مع القواعد والقاموس المضغوط)؛ الإضافة وPWA تفحصان محليًا في ذاكرة المتصفح: خصوصية مطلقة وزمن صفري شبكيًا، ويُستدعى الخادم فقط للطبقات الثقيلة (صرف كامل/عصبي).
- بايثون يبقى المرجع (reference implementation) ومختبر القواعد؛ التكافؤ يُثبت بالمجموعة الذهبية في CI للطرفين.

## V2-م4 — التعاون الفوري وCRDT

- طبقة مزامنة نصية بـ CRDT (Yjs/Yrs أو تنفيذ RGA خاص فوق نواة Rust) — المواضع تصبح معرفات CRDT مستقرة لا offsets، فتنجو الملاحظات من التحرير المتزامن.
- خادم مزامنة WebSocket بلا حالة دائمة (غرف في Redis)، والتشفير طرف-إلى-طرف خيار أول التصميم لأن المحرك يعمل محليًا عبر WASM — الخادم يمرر تحديثات معماة ولا يرى النص.
- إعادة تعريف محدد المعدل والجلسات كخدمات موزعة (Redis) لإزالة آخر حالة داخل العملية.

## V2-م5 — الدمج السياقي بالمحولات (تجاوز n-grams)

- نموذج عربي مضغوط (distilled، 4-bit، هدف < 150MB) للأخطاء السياقية real-word وWSD، خلف نفس `NeuralBackend` Protocol — **بوابة الثقة لا تتغير**: الحتمي يتكلم أولًا، والعصبي مرشَّح ضد التداخل، وis_auto_applied=False دائمًا.
- تجميع طلبات (batching) وتخزين نتائج على مستوى الجملة، وقياس زمن استدلال منشور لكل عتاد.
- أتمتة خط «التقطير إلى قواعد»: التصحيحات العصبية المقبولة تتحول آليًا إلى قواعد YAML مع التحقق الذاتي الحالي — النموذج يكبر والخندق الحتمي يكبر معه.
- استنساخ الصوت: توليد إعادة الصياغة مشروطًا ببصمة V2-م1 (المستخدم يرى «بصوتك أنت»)، محليًا أو عبر backend يختاره المستخدم، ولا يُطبَّق تلقائيًا أبدًا.

## V2-م6 — منصة التوزيع والقياس

- نشر الإضافة في متاجر Chrome/Edge/Firefox، ولوحة مفاتيح محمولة فوق نواة WASM.
- corpus طبيعي بمراجعة بشرية مزدوجة (بروتوكول `ANNOTATION_GUIDELINES_AR.md` موجود أصلًا) لإحلال ادعاء دقة عام محل معيار القوالب.
- OpenTelemetry، p50/p99 منشورة، SLO، اختبار اختراق مستقل، SBOM موقّع.

## ترتيب التنفيذ ولماذا

م1 الآن لأنها قيمة مستخدم فورية بلا أي إعادة هيكلة وخطرها صفر على النواة؛ م2 قبل م3 لأن نقل نواة بطيئة التصميم إلى Rust يخلّد أخطاءها؛ م3 قبل م4 لأن التعاون المشفر طرف-إلى-طرف يفترض محركًا يعمل عند الطرف؛ م5 بعد م3 لأن التقطير يحتاج خط قواعد مستقرًا عبر اللغتين؛ م6 حين يصبح لدينا ما يستحق التوزيع الواسع.
