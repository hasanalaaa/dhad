# تقرير Dhad vMAX — البوابة صفر والمرحلة الأولى

تاريخ القياس: 2026-07-22. البيئة المرجعية: Apple M3 Max، Python 3.14.6، Rust 1.97.1، Node 26.5.0.

## ما نُفذ

- أزيل تضارب `0.13.0` من Docker وCompose، وأصبح `src/dhad/__init__.py` مصدر إصدار Python الوحيد عبر Hatch dynamic version.
- استُبدل Uvicorn worker المهمل بالحزمة المستقلة `uvicorn-worker`، وثُبت بناء wheel وتثبيتها من wheelhouse مع extras الخادم والإنتاج.
- أضيف dependency lock للاختبار وCI مستقل لـPython وRust.
- أصبح حل تعارض المقاطع `O(n log n)`: AVL في Python و`BTreeMap` في Rust، مع اختبارات تفاضلية ضد المرجع التربيعي.
- استُبدل مسح 130 قاعدة حرفية بمسح Aho–Corasick واحد في Python وRust، مع الحفاظ على حدود الكلمات وترتيب القواعد والاستثناءات والتكافؤ.
- أضيف `AnalysisContext` غير قابل للتغيير ليحمل token stream وnormalized lookup forms وفهرس الجمل وsyntax parse لمراجعة واحدة من المستند.
- تشترك القواعد والإملاء واللهجات والدلالة والأسلوب في السياق نفسه داخل `Dhad.check`، ويعاد استخدام parse واحد بعد neural refinement.
- أصبح تقسيم السطور كثيفة النقاط خطياً بدلاً من إعادة البحث من بداية السطر عند كل نقطة.
- أصبح فحص تداخل PII لوغاريتمياً، وانتقلت استعادة آلاف المقاطع المحمية من استبدالات متكررة إلى مسح خطي واحد.
- أصلحت جميع مخالفات `cargo clippy -D warnings` ووثقت عقود أمان واجهة WASM الحالية، ثم نُسقت الشيفرة بـ`cargo fmt`.

## بوابات الجودة

- Python: `1767 passed, 1 skipped` مع تحويل التحذيرات إلى أخطاء.
- Rust: 15 اختباراً، و`cargo fmt --check` و`cargo clippy --all-targets -- -D warnings` نظيفان.
- JavaScript: جميع الملفات اجتازت `node --check`.
- WASM: تطابق 10/10 مستندات golden مع مرجع Python.
- Wheel: بُنيت `dhad-1.0.0rc1` وثُبتت في بيئة نظيفة بلا شبكة من wheelhouse، ثم نجح استيراد FastAPI ذي 22 route.

## الحد الفاصل للمرحلة الثانية

لم تُنفذ بعد واجهة WASM packed/zero-copy أو persistent document handles أو `wasm-opt` أو فصل القاموس المضمّن؛ هذه أعمال المرحلة الثانية، ولن تُخلط مع نجاح المرحلة الأولى.
